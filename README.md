# ğŸ¾ VetClinic Manager

**VetClinic Manager** is a robust web system designed for veterinary clinic management.
The project centralizes the control of pet owners, animals, clinical staff, appointments, and immunization records within an intuitive and responsive interface.

---

## ğŸ“ About the Development

This software was developed as a practical project, combining custom backend logic with a modern user interface.

* **Backend:** Server-side logic and routes were developed in Python using Flask, based on a logical structure provided by **Gemini AI** and implemented by the developer.
* **Database:** The system uses **raw SQL (MySQL)** for data manipulation, without ORMs (such as Flask-SQLAlchemy), ensuring maximum performance and full control over SQL queries.
* **Frontend:** The visual interface and all HTML templates were generated by **Gemini AI**, using HTML5, Bootstrap 5, and FontAwesome icons.

---

## ğŸš€ Technologies Used

* **Language:** [Python 3.13.11](https://www.python.org/)
* **Web Framework:** [Flask](https://flask.palletsprojects.com/)
* **Database:** [MySQL](https://www.mysql.com/) (pure SQL queries)
* **Interface:** [Bootstrap 5.3](https://getbootstrap.com/) & [FontAwesome](https://fontawesome.com/)

---

## ğŸ“Š Database Structure

The database architecture was designed to ensure data integrity and long-term persistence of clinical history:

* **Client:** Pet owner data with integrity enforced via CPF.
* **Pet:** Animal records linked to owners, using `ON DELETE SET NULL`.
* **Veterinarian:** Identification via CRMV with `ON UPDATE CASCADE`.
* **Appointment:** Appointment management with status control (Scheduled, In Progress, Completed) and medical records.
* **Vaccine:** Immunization history linked to each pet.
